## 3. Architecture

**Core Stack:**
- **Containers:** Docker or Kubernetes (target workloads).
- **Metrics:** Prometheus + cAdvisor (Docker) or kube-state-metrics + node-exporter (K8s).
- **Visualization & Alerting:** Grafana (dashboards, Alertmanager integration).
- **AI Engine:** Claude (Sonnet/Opus via Anthropic API) orchestrated by Claude Code CLI.
- **Workflow Layer:** Python-based workflows (orchestrated via simple scheduler, webhooks, or lightweight Prefect/Argo if scaled). Claude Code CLI used for development and runtime agentic tasks.
- **Action Layer:** Kubernetes Python client or Docker SDK for safe mutations (scale, restart, patch resources).
- **Integration:** Prometheus HTTP API / prometheus-api-client, Grafana HTTP API, Anthropic SDK.

**Data Flow:**
1. Prometheus scrapes container metrics continuously.
2. Workflow triggered (scheduled / Alertmanager webhook / manual).
3. Prom client fetches relevant time-series.
4. AI analyzer (Claude) receives structured metrics + context → returns JSON + natural language explanation + recommended actions + confidence score.
5. Decision engine applies guardrails (dry-run mode, approval queue, confidence threshold).
6. Action executor performs safe changes or creates GitOps PR.
7. Grafana integrator posts annotations, updates panels, or creates dynamic alert rules.
8. Results logged + fed back to Prometheus (custom metrics for AI decisions) and Grafana.

**Deployment Options:**
- Local dev: docker-compose (full stack + workflow app).
- Production: Kubernetes DaemonSet/sidecar or dedicated Deployment + ServiceAccount with limited RBAC.
- CI/CD: GitHub Actions or GitLab CI using Claude Code in non-interactive mode.

## 4. Detailed Sequence of Implementation Steps

### Phase 1: Foundation (1-2 days)

**Step 1.1: Infrastructure & Monitoring Stack**
- Install Docker (or minikube/kind for K8s).
- Create `docker-compose.yml` with:
  - prometheus:latest
  - grafana:latest
  - cadvisor (for Docker container metrics)
  - node-exporter (host metrics)
  - Optional: Alertmanager
- Configure Prometheus scrape jobs for cadvisor (port 8080) and node-exporter.
- Start stack: `docker compose up -d`
- Verify: Prometheus targets page shows healthy, Grafana at http://localhost:3000 (admin/admin). Import "Docker Container Overview" or "Kubernetes" community dashboards.
- Expose Prometheus at http://prometheus:9090 inside network.

**Step 1.2: Claude Code CLI Installation & Authentication**
- Run: `curl -fsSL https://claude.ai/install.sh | bash`
- Set environment: `export ANTHROPIC_API_KEY=sk-ant-...`
- Verify: `claude --version` and `claude -p "Hello, test project context"`
- (Optional) Configure global settings or project-specific via `.claude/` files.

**Step 1.3: Project Scaffolding (Use Claude Code)**
- `mkdir container-obs-ai-workflows && cd container-obs-ai-workflows`
- Launch interactive: `claude`
- Initial prompt (paste this):
  ```
  Initialize a professional Python project for AI-powered container observability workflows.
  Tech: Prometheus (metrics), Grafana (dashboards/alerts), Claude via Anthropic API, Kubernetes/Docker actions.
  Create full folder structure, requirements.txt, README.md, config management (YAML + env), logging, error handling, and safety guardrails.
  Include modules for: anomaly detection, predictive scaling, auto-remediation, Grafana integration.
  Use best practices, type hints, async where useful, and comprehensive docstrings.
  ```
- Claude Code will create the skeleton. Review diffs, approve file creations/edits.
- Run `claude "Generate .env.example and settings.yaml with placeholders for thresholds and API keys."`

### Phase 2: Core Workflow Development (3-5 days) – Iterative with Claude Code

Use Claude Code heavily here. Work in interactive mode or one-shot with `-p`. Always review before applying changes. Commit frequently.

**Step 2.1: Prometheus Client & Data Layer**
- Prompt: `claude "Create utils/prom_client.py with robust PrometheusConnect wrapper. Support custom queries for container_cpu_usage_seconds_total, memory, network, filesystem. Add retry logic, caching (TTL 30s), and structured output (pandas DataFrame or dict). Handle multi-container and namespace filtering."`
- Test queries against running Prometheus.

**Step 2.2: AI Analyzer Core**
- Prompt: `claude "Create utils/ai_analyzer.py. Function analyze_container_metrics(metrics_dict, container_name, time_window) that builds a detailed prompt for Claude, calls Anthropic Messages API (model claude-3-5-sonnet-20241022), and parses response into strict JSON: {severity, anomaly_detected, explanation, root_cause, recommended_actions (list), confidence (0-1), suggested_grafana_annotation}. Add system prompt for safety and container domain expertise. Include token usage logging."`
- Add support for few-shot examples in prompt for better JSON adherence.

**Step 2.3: Anomaly Detection Workflow**
- Prompt: `claude "Implement workflows/anomaly_detector.py. Scheduled or webhook-triggered function that: fetches last 5-15min metrics for target containers via prom_client, runs ai_analyzer, if anomaly: logs + calls grafana_integrator to annotate dashboard + optionally triggers remediation. Support batch processing for multiple containers. Add CLI entrypoint with argparse."`
- Add statistical pre-filter (e.g., z-score or simple threshold) before expensive Claude call to control costs.

**Step 2.4: Predictive Scaling Workflow**
- Prompt: `claude "Create workflows/predictive_scaler.py. Use simple moving average or statsmodels/Prophet (add to requirements if needed) for short-term forecast of CPU/memory. Feed forecast + current metrics to ai_analyzer for scaling decision (scale up/down, by how much, target replicas). Output safe recommendation only (never auto-apply without flag). Integrate with Kubernetes client for dry-run patch."`
- Guardrail: Max scale step = +2 replicas, min replicas respected.

**Step 2.5: Auto-Remediation Workflow**
- Prompt: `claude "Implement workflows/auto_remediator.py with safety-first design. Supported actions: restart_container/pod, scale_deployment, update_resource_limits (with validation). Require confidence > 0.85 and explicit --auto flag or human approval queue (simple file or Redis). Use kubernetes client (or docker SDK for pure Docker). Log every action to Prometheus custom metric and Grafana annotation. Never delete resources."`
- Add circuit breaker: pause remediation for namespace if >3 actions in 10min.

**Step 2.6: Grafana Integration**
- Prompt: `claude "Create utils/grafana_integrator.py. Functions: add_dashboard_annotation(dashboard_uid, text, tags), create_or_update_alert_rule, post_ai_insight_panel_data. Use Grafana HTTP API (requests + auth token). Support both admin API key and service account. Make idempotent."`
- Bonus: Dynamic panel creation for AI confidence scores.

**Step 2.7: Orchestrator & CLI**
- Prompt: `claude "Create main.py as unified CLI and entrypoint. Subcommands: detect, scale, remediate, full-workflow, status. Support --dry-run, --container, --namespace, --auto flags. Add simple scheduler loop (or integrate APScheduler). Expose health endpoint if run as service."`
- Prompt: `claude "Add webhook server (FastAPI) in main.py to receive Alertmanager webhooks and trigger relevant workflow."`

**Step 2.8: Configuration, Logging, Safety**
- Prompt: `claude "Enhance config/settings.yaml with sections for thresholds (cpu_anomaly_threshold, etc.), ai (model, max_tokens, temperature=0 for determinism), actions (dry_run_default, approval_mode, max_actions_per_hour), grafana (url, token). Use pydantic-settings or similar. Add structured logging (structlog or loguru) with JSON output for easy ingestion."`
- Prompt: `claude "Add comprehensive safety module: validate_action_safety(), rate_limiter, audit_logger. All actions must pass safety checks."`

## 6. Technology Choices & Justifications

- **Python 3.11+**: Rich ecosystem (prometheus-api-client, kubernetes, anthropic, fastapi, pydantic).
- **Claude Code CLI**: Primary development agent – accelerates coding 5-10x, understands full codebase context, handles git, tests, refactoring.
- **Anthropic SDK direct calls**: For runtime AI inference (Claude Code is dev-time primarily).
- **Prometheus + Grafana**: Industry standard, already likely in use; excellent APIs.
- **Minimal orchestration**: Start simple (CLI + cron/webhook). Scale to Temporal/Prefect later if needed.
- **Safety-first design**: Dry-run default, confidence gates, rate limits, audit everything.

## 7. Risks, Mitigations & Cost Control

**Risks:**
- High Claude token costs on frequent/large queries.
- AI false positives/negatives leading to unnecessary actions or missed issues.
- Security: Over-privileged service accounts or leaked keys.
- Prompt injection or model drift.

**Mitigations:**
- Pre-filter with cheap stats (stddev, thresholds) → only call Claude on suspicious cases.
- Temperature=0, strict JSON mode, few-shot examples, Pydantic validation.
- All actions default dry-run; production requires explicit --auto and human gate initially.
- Dedicated low-privilege K8s ServiceAccount + Kyverno/OPA policies if available.
- Cost dashboard using existing Claude Code Grafana plugin + custom token metrics.
- Fallback to rule-based alerts if AI unavailable.

**Cost Estimate (MVP, moderate usage):**
- Claude API: $50-200/month depending on query volume and model.
- Infrastructure: Negligible (existing Prom/Grafana + small workflow pod).
